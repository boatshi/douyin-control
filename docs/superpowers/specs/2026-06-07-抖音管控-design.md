# 📱 抖音学习管控 App — 设计文档

> 为孩子（大班→六年级）定制的 Android 平板学习激励工具
> 做题赚积分 → 积分换抖音观看时间
> 原生 Android 开发

---

## 一、项目概述

### 1.1 项目背景

孩子每天想看抖音短视频，但需要建立"先学习后娱乐"的习惯。通过 App 内嵌 douyin.com 和做题系统，配合华为平板的儿童模式锁定，实现：

- **做题赚积分** → 答对给分、答错不给分
- **积分换时间** → 用积分兑换抖音观看时长
- **错题复习** → 自动收录错题，重新做对也给积分
- **AI 出题** → 实时生成不重复题目，覆盖大班到六年级

### 1.2 技术方案

| 项目 | 选择 |
|------|------|
| 架构 | 原生 Android App |
| 语言 | Kotlin |
| UI | Jetpack Compose |
| 数据库 | SQLite（Room ORM） |
| 网络 | OkHttp / Retrofit |
| AI 接口 | DeepSeek API（通过 API 代理） |
| 嵌入式内容 | WebView（douyin.com，桌面 UA） |
| 系统管控 | 华为平板儿童模式/健康使用平板 |
| 开发工具 | Android Studio |

### 1.3 为什么选原生 Android

- 体验最好：原生 WebView、SQLite 调用零延迟
- 后台计时可靠：Service 保活
- 本地运行：不需服务器，离线可做题（种子题库）
- Jetpack Compose 现代 UI 开发效率高

## 二、界面风格

### 2.1 整体视觉
- **风格**：卡通可爱，大色块，大按钮
- **配色**：蓝紫渐变（#667EEA → #764BA2） — 延续云帆乐园风格
- **元素**：大圆角按钮（48px+）、柔和阴影、明亮卡片
- **字体**：大号圆润字体，适合儿童阅读
- **图标**：Emoji 图标引导，降低识字门槛

### 2.2 设计原则
- 所有按钮至少 48dp 高度，方便小手点击
- 核心操作配图标 + 文字双重引导
- 反馈清晰：答对撒花 🎉，答错温和提示正确答案
- 减少文字量，多用图形化表达
- 所有列表支持翻页（每页固定条数）

### 3.2 📚 做题模式

**流程：**
```
进入做题
  → 选择科目（数学/语文）
  → 当前年级（家长设置中配置）
  → 加载一道题（选择题，4选1）
  → 孩子选择答案
    → 答对：🎉 加分 + 自动下一题
    → 答错：😊 显示正确答案 + 自动收入错题库 + 下一题
  → 随时可点击"退出"返回首页（积分已累积的不丢失）
```

**题目来源优先级：**
1. 本地 SQLite 题库中取已存在的题目
2. 题目不够时 → 调用 DeepSeek API 生成新题 → 去重后存入本地
3. 无网络时 → 只用本地已有题目（首次安装有种子题库）

**翻页方式：**
- 每题一页，天然翻页（上一题/下一题）
- 顶部显示进度：第 3/10 题
- 做过的题不可回退修改

**积分结算规则：**
| 情况 | 积分 |
|------|:----:|
| 答对一题 | +10（默认，可调） |
| 答错一题 | 0 |
| 错题重新做对 | +10（同正常题） |

**翻页：**

**数据库字段：** 错题在 `study_logs` 表中标记，不单独建表，通过 `result=wrong + question_id` 查询

**进入条件：** 积分足够 → 扣除积分 → 进入抖音界面

**界面：**

│                              │
│  剩余可看：████████░░░░ 8:32 │
└──────────────────────────────┘

**功能点：**
- WebView 加载 `douyin.com`，User-Agent 设为桌面版 Chrome
- 顶部固定栏显示剩余可看时间（实时倒计时）
- 时间归零 → 弹窗提示"时间到了，去做题赚积分吧！" → 自动返回首页

- 退场时记录本次实际观看时长到 study_logs

**技术要点：**
- WebView 启用 JavaScript + DOM Storage
- 禁用 WebView 内链接打开外部浏览器
- 注入 JS 检测页面加载状态
- 支持触摸滑动

## 四、数据模型

### 4.1 题目表 (questions)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| subject | TEXT | 科目：math / chinese |
| grade | TEXT | 年级：bigclass, grade1 ~ grade6 |

| deleted | INTEGER | 0=正常, 1=软删除(去重过滤) |
| source | TEXT | seed / ai / manual |
| created_at | INTEGER | 创建时间戳 |

**去重索引：** 对 `question` 字段建立唯一索引或程序级别去重

### 4.2 积分规则表 (rules)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| rule_key | TEXT | 规则标识（唯一） |
| rule_name | TEXT | 规则名称 |
| value | INTEGER | 积分值 |
| updated_at | INTEGER | 最后修改时间戳 |

**种子数据：**
| rule_key | rule_name | value |
|----------|-----------|:-----:|
| score_per_question | 做对一题得分 | 10 |
| score_per_rework | 错题重做对得分 | 10 |

### 4.3 积分流水表 (points_log)

| 字段 | 类型 | 说明 |
|------|------|------|

| source_detail | TEXT | 具体描述，如"做对数学题：8+5=?" |

### 4.4 学习记录表 (study_logs)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| type | TEXT | question / video |

| result | TEXT | correct / wrong / rework_correct |

## 五、AI 出题与去重

### 5.1 触发时机

| 场景 | 说明 |
|------|------|
| 启动时 | 检查题库，若当前年级题目 < 50 道 → 后台自动补充 |
| 做题时 | 当前年级题目用完 → 触发 AI 补充 |
| 家长手动 | 家长在设置页主动点击"AI 出题" |

### 5.2 去重逻辑

```
AI返回题目列表
  → 逐题检查 question 字段
    → 精确匹配数据库已有 → 标记重复，丢弃
    → 相似度匹配（Levenshtein距离 < 3） → 标记重复，丢弃
    → 无重复 → 存入数据库
  → 若丢弃后有效题目 < 请求数量
    → 继续请求AI补充，直到够数或AI无新题可出
```

### 5.3 API 提示词模板

```
你是一位小学教育专家，请为{年级}学生出{数量}道{科目}选择题。
每道题4个选项，仅一个正确答案。
以JSON格式返回，不要返回其他内容：
[
  {{
    "question": "题目内容",
    "options": ["A选项", "B选项", "C选项", "D选项"],
    "answer": "正确答案"
  }}
]
```

### 5.4 种子题库

首次安装时，内置 100+ 道种子题目（大班~二年级为主），保证无网络时也可做题。

## 八、核心流程汇总

### 8.1 做题赚积分流程

```
用户进入做题 → 选择科目 → 展示题目（4选1）
  → 用户选择答案
    → 正确 → 积分+10 → 记录 study_logs(correct)
            → 如果本题在错题库中 → 移出错题库 → 记录 rework_correct
    → 错误 → 记录 study_logs(wrong) → 自动加入错题库
            → 显示正确答案 → 停顿1.5秒 → 自动下一题
  → 用户可随时退出做题（返回首页）
```

### 8.2 看抖音扣积分流程

```
点击"看抖音"
  → 查询当前积分 ≥ min_cost_threshold ?
    → 否 → 弹窗"积分不足，去做题吧"
    → 是 → 计算积分可兑换时长（积分 ÷ cost_per_minute）
         → 扣除全部积分？按需扣除（比如每次最多扣30分钟的量）
         → 打开 WebView 加载 douyin.com（桌面UA）
         → 倒计时运行（TimerEngine）
         → 时间到 / 用户退出
           → 记录 study_logs(video, 实际时长)
           → 返回首页，剩余时间下次可用（未扣完的积分退还？）
```

**积分扣除策略：** 每次看抖音，按扣除当前积分可兑换的最大时长。例如积分 100，每分钟消耗 10 分，则可看 10 分钟。一次性扣除（100 分），若提前退出，按比例退还剩余时长的积分。

### 8.3 AI 出题补题流程

```
检查当前年级题库数量 → < 50 题？
  → 是 → 调用 DeepSeek API（年级+科目+数量）
  → 返回题目 JSON
  → 逐题去重
    → 重复 → 丢弃
    → 不重复 → 存入 SQLite
  → 有效题目 < 请求数量？
    → 继续请求补充
  → 完成
```

### 11.2 WebView 抖音优化

```kotlin
// 关键配置
webView.settings.apply {
    javaScriptEnabled = true
    domStorageEnabled = true
    userAgentString = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ..."
    mediaPlaybackRequiresUserGesture = false
    mixedContentMode = WebSettings.MIXED_CONTENT_ALWAYS_ALLOW
}
// 拦截链接：禁止跳转到外部浏览器
webView.webViewClient = object : WebViewClient() {
    override fun shouldOverrideUrlLoading(view: WebView, request: WebResourceRequest): Boolean {
        val url = request.url.toString()
        return if (url.startsWith("http")) {
            false // 在WebView内加载
        } else {
            true // 拦截非http链接（如intent://）
        }
    }
}
```
